# MSW Browser Worker Setup

> Last updated: 2026-05-25

## Overview

The frontend uses [Mock Service Worker (MSW)](https://mswjs.io/) to intercept API calls in two contexts:

| Context | File | MSW API |
|---|---|---|
| **Browser (dev)** | `src/mocks/browser.ts` | `setupWorker` from `msw/browser` |
| **Tests (Vitest)** | `src/tests/mocks/server.ts` | `setupServer` from `msw/node` |

The same handler arrays (e.g. `referenceHandlers`, `formsHandlers`) are imported by **both** files — MSW automatically uses the right implementation per environment.

---

## ⚠️ Runtime vs Test-Only — Governance Rule

> ❌ **Business verticals MUST NOT use runtime MSW after their backend exists.**

Once a module has Laravel controllers, migrations, seeders, and API routes, its handlers
MUST be removed from `src/mocks/browser.ts` and used only in Vitest.

### Which modules may use runtime MSW?

| Module | Runtime MSW | Reason |
|--------|:-----------:|--------|
| Reference (cookbook) | ✅ | Frontend-only demo/sandbox |
| Dynamic Forms | ✅ | Transitional — forms builder is frontend-heavy |
| **CondoFlow** | ❌ | Real backend vertical (removed 2026-05-25) |
| **MiniHIS** (future) | ❌ | Will follow CondoFlow architecture |
| Any future business vertical | ❌ | Must use real APIs after backend exists |

### Validating real backend usage

After removing runtime MSW for a module:

1. DevTools Network tab shows real XHR/fetch (no "from service worker")
2. Laravel Telescope captures the requests
3. Responses contain seeded database data
4. Sanctum cookies (`XSRF-TOKEN`, `laravel_session`) are sent correctly

---

## How It Works in Development

1. `main.ts` detects `import.meta.env.DEV === true`
2. Dynamically imports `src/mocks/browser.ts`
3. Calls `worker.start({ onUnhandledRequest: 'bypass' })`
4. MSW registers a Service Worker (`public/mockServiceWorker.js`)
5. All `fetch()` calls made by `apiClient` are intercepted by the registered handlers
6. Requests with no matching handler fall through to the Vite proxy (→ Laravel)

---

## Service Worker File

`public/mockServiceWorker.js` was generated by:
```bash
npx msw init public/ --save
```

This file must be committed. It is referenced by `msw.workerDirectory` in `package.json`.  
**Do not edit this file manually.** Re-run the init command after upgrading MSW.

## Adding Handlers for a New Module (Demo/Reference Only)

### Step 1 — Create the handler file

```ts
// src/modules/my-module/mocks/handlers.ts
import { http, HttpResponse } from 'msw'

export const myModuleHandlers = [
  http.get('/api/my-module/items', () =>
    HttpResponse.json({ data: [] }),
  ),
]
```

### Step 2 — Register for browser dev

```ts
// src/mocks/browser.ts
import { myModuleHandlers } from '@/modules/my-module/mocks/handlers'

export const worker = setupWorker(
  ...devAuthHandlers,
  ...referenceHandlers,
  ...formsHandlers,
  ...myModuleHandlers,  // ← add here
)
```

### Step 3 — Register for Vitest

```ts
// src/tests/mocks/server.ts
import { myModuleHandlers } from '@/modules/my-module/mocks/handlers'

export const server = setupServer(
  ...authHandlers,
  ...formsHandlers,
  ...referenceHandlers,
  ...myModuleHandlers,  // ← add here
)
```

## Dev Auth User

`src/mocks/browser.ts` provides a pre-authenticated dev user so you can navigate the app without a real Laravel session:

```ts
const mockDevUser = {
  id: 1,
  name: 'Dev User',
  email: 'dev@example.com',
  is_platform_admin: true,   // ← platform admin bypasses all permission checks
}
```

To simulate a restricted user, temporarily change `is_platform_admin: false` and add specific permissions.

## Debugging MSW in the Browser

1. Open DevTools → **Application** → **Service Workers** — confirm `mockServiceWorker` is active
2. Open the **Network** tab — MSW-intercepted requests show `(ServiceWorker)` in the Initiator column
3. Check the browser Console — MSW logs every intercepted request by default

To disable MSW for one session: `localStorage.setItem('msw:disabled', 'true')` and reload.

## onUnhandledRequest: 'bypass'

The worker is started with `onUnhandledRequest: 'bypass'`. This means:
- Requests to `/api/reference/*` → intercepted by MSW ✅
- Requests to `/api/dynamic-forms/*` → intercepted by MSW ✅  
- Requests with no handler → forwarded to Vite proxy → Laravel ✅ (no warning)

Change to `'warn'` temporarily to discover unhandled routes during development.
